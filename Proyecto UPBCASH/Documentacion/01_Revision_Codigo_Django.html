Revision Tecnica de Codigo Django - UPBCash

Proyecto: UPBCash Django

Fecha: 2026-02-22

Alcance: Repositorio UPBCash/Proyecto UPBCASH/upbcash_django

Resumen ejecutivo

Se realizo revision tecnica backend, se identificaron riesgos de seguridad y control de acceso, y se implementaron correcciones criticas sin afectar compatibilidad local ni contratos API existentes.

Hallazgos principales

ID Hallazgo Severidad Estado
SEC-001 Las rutas /administrador/ y /administrador/mapa/ permitian acceso con solo autenticacion. Critica Corregido
SEC-002 Configuracion de despliegue reportaba advertencias de seguridad en check --deploy (HSTS/SSL cookies/DEBUG/SECRET_KEY). Alta Corregido

Cambios implementados

Archivo Cambio Impacto
core/views.py Se agrego helper de autorizacion administrativa y se aplico en admin_inicio y admin_mapa. Bloquea acceso administrativo a usuarios no superusuario.
upbcash/settings.py Se agregaron settings de seguridad por entorno: SECURE_SSL_REDIRECT, SESSION_COOKIE_SECURE, CSRF_COOKIE_SECURE, SECURE_HSTS_SECONDS, SECURE_HSTS_INCLUDE_SUBDOMAINS, SECURE_HSTS_PRELOAD. Mejora postura de seguridad en produccion.
upbcash/settings.py Se agrego validacion obligatoria de fortaleza de DJANGO_SECRET_KEY cuando DJANGO_DEBUG=False. Evita despliegues inseguros con secret key debil o default.
core/tests.py Se agregaron pruebas de autorizacion para panel admin. Cubre regresion de acceso indebido.

Variables de entorno nuevas

  • DJANGO_SECURE_SSL_REDIRECT
  • DJANGO_SESSION_COOKIE_SECURE
  • DJANGO_CSRF_COOKIE_SECURE
  • DJANGO_SECURE_HSTS_SECONDS
  • DJANGO_SECURE_HSTS_INCLUDE_SUBDOMAINS
  • DJANGO_SECURE_HSTS_PRELOAD

Conclusiones

Se mitigaron los riesgos criticos detectados. El backend mantiene estabilidad funcional y mejora su perfil de seguridad para despliegue.
